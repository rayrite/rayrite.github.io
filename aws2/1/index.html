Chapter 1

Basic AI Concepts and Terminology

Exam Focus: Understanding the differences between AI, ML, and Deep Learning, as well as the various learning paradigms (supervised, unsupervised, reinforcement).

Core AI Definitions

Artificial Intelligence (AI): Computer systems capable of performing tasks that typically require human intelligence, including visual perception, speech recognition, decision-making, and language translation.

AI vs ML vs Deep Learning

Concept Definition Relationship
AI Broad field of creating intelligent systems Umbrella term
ML Algorithms that learn from data Subset of AI
Deep Learning Neural networks with multiple layers Subset of ML

Learning Paradigms

📊 Supervised Learning

Training on labeled data (input-output pairs).

Use Cases: Fraud detection, image classification, price prediction

🔍 Unsupervised Learning

Finding structure in data without labels.

Use Cases: Customer segmentation, anomaly detection

🎮 Reinforcement Learning

Learning through interaction and feedback (rewards/penalties).

Use Cases: Game playing, robotics, autonomous systems

🔄 Self-Supervised Learning

Model generates its own labels from data structure.

Examples: BERT, GPT pre-training

Key Historical Milestones

Year Milestone
1956 Dartmouth Conference - AI term coined
1980s Backpropagation algorithm; Hopfield Network
2012 AlexNet breakthrough (deep learning)
2017 Transformer architecture introduced
2024 Nobel Prize in Physics for ML (Hopfield & Hinton)

Chapter 2

Basic Concepts of Generative AI

Exam Focus: Understanding tokens, embeddings, transformer architecture, and the differences between foundation models and other AI approaches.

Generative AI Fundamentals

Generative AI: AI systems that create new content (text, images, audio, video) rather than just classifying or predicting.

Core Concepts

Concept Definition Example
Tokens Atomic units of text "Tokenization" → ["Token", "ization"]
Embeddings Vectors representing tokens in high-dimensional space "king" and "queen" cluster together
Chunking Grouping related tokens "Gen AI" as single concept

Transformer Architecture

Key Innovation: Self-attention mechanism from "Attention Is All You Need" (2017) - each token "pays attention" to every other token, capturing long-range dependencies.

How Self-Attention Works

  1. Attention Weights: Calculate relevance between all token pairs
  2. Summation: Combine weighted signals for context
  3. Multiple Heads: Different heads focus on different aspects

Foundation Models

Large, pre-trained models serving as base for various tasks:

  • GPT family: GPT-4, GPT-3.5
  • Anthropic: Claude
  • Amazon: Nova, Titan
  • Meta: Llama 3, Llama 2

Multi-modal Models

Model Type Description
Vision Transformers (ViT) Process images as patches (tokens)
Audio Transformers Process audio segments as tokens
Multi-modal Transformers Handle text, images, audio together

Diffusion Models

Learn by adding noise to data and reversing the process:

  • More stable training than GANs
  • Applications: Image generation, editing, enhancement
  • Examples: Stable Diffusion XL, DALL·E

Model Selection and Prompt Engineering

Exam Focus: Understanding inference parameters (temperature, top-p, top-k) and prompt engineering techniques.

Model Selection Criteria

Factor Considerations
Task Type Text generation, summarization, classification, code
Performance Accuracy, latency, throughput requirements
Cost Per-token pricing, provisioned throughput options
Context Window Max input/output tokens needed

Model Categories in Bedrock

Category Best For
Amazon Nova General purpose, cost-effective
Anthropic Claude Reasoning, safety, long context
Meta Llama Open-weight, customizable
Stability AI Image generation

Prompt Engineering

Core Components

Component Purpose Example
Instruction Clear task direction "Summarize the following"
Context Background information "As a financial advisor..."
Examples Demonstrate desired output "Input: X, Output: Y"
Constraints Limit the response "in 50 words or less"
Format Specify output structure "as a bullet-point list"

Prompt Patterns

Pattern Description
Zero-shot No examples provided
Few-shot Provide 2-3 examples
Chain-of-Thought Guide step-by-step reasoning
System Prompting Set global behavior/context

Inference Parameters

Temperature

  • 0.0-0.3: Deterministic, focused
  • 0.4-0.7: Balanced creativity
  • 0.8-1.0+: Highly creative

Top-p (Nucleus)

  • Low: Conservative selection
  • Medium: Balanced
  • High: More diversity

Chapter 6

Generative AI Applications with RAG and Agents

Exam Focus: Understanding RAG architecture, Bedrock Knowledge Bases, and Agents capabilities.

Retrieval-Augmented Generation (RAG)

Problem Solved: LLMs have knowledge cutoff and can't access private/updated data. RAG enhances responses with retrieved external information.

RAG Workflow

  1. User Query - Input question or request
  2. Retrieve Documents - Find relevant information from knowledge base
  3. Augment Prompt - Add retrieved context to original prompt
  4. Generate Response - Foundation model generates grounded answer

RAG Components

Component Purpose AWS Service
Vector Store Store document embeddings OpenSearch, Aurora, RDS
Embedding Model Convert text to vectors Titan Embeddings
Retriever Find relevant documents Bedrock Knowledge Bases

Bedrock Knowledge Bases

  • Fully managed RAG: Automated ingestion, retrieval, prompt augmentation
  • Data Sources: S3, databases, document repositories
  • Features: Automatic citations, multi-modal support

Amazon Bedrock Agents

Agent Capabilities

Capability Description
Action Groups Define APIs the agent can call
Knowledge Bases Integrate RAG for contextual responses
Session Management Maintain conversation state
Planning Multi-step reasoning and task decomposition

Agent Architecture

User Input → Agent → Foundation Model → Action Planning 
                                    ↓
                           Knowledge Base (RAG)
                                    ↓
                           API Execution → Response

Bedrock Data Automation

Capability Description
Document Processing Extract structured data from documents
Video Analysis Generate summaries, detect events
Image Analysis Content detection, inappropriate content filtering
Visual Grounding Link outputs to source locations

Chapter 7

Model Customization and Evaluation

Exam Focus: Understanding fine-tuning, continued pre-training, distillation, and model evaluation approaches.

Model Customization Techniques

Fine-tuning

Attribute Description
Definition Further training on domain-specific labeled data
Data Required Input-output pairs relevant to task
Benefit Improved accuracy for specific use cases
Use Cases Specialized terminology, brand voice, specific tasks

Continued Pre-training

Attribute Description
Definition Training on unlabeled domain data to expand knowledge
Data Required Large corpus of domain text
Benefit Better understanding of specialized vocabulary

Model Distillation

Attribute Description
Definition Transfer knowledge from large model to smaller model
Benefit Faster inference, lower cost, smaller footprint
Trade-off Some accuracy loss vs. original large model

Custom Model Import

  • Bring your own models to Bedrock
  • Serverless hosting with Bedrock ecosystem integration
  • Use Case: Proprietary models, fine-tuned open-source models

Model Evaluation

Automated Evaluation

Metric Description
Accuracy Correctness of responses
Robustness Performance under perturbations
Toxicity Harmful content detection
Semantic Similarity Meaning preservation

Human Evaluation

  • Custom metrics for domain-specific criteria
  • Rating scales (1-5 or custom)
  • Workforce options: AWS-managed or bring your own
  • Qualitative feedback and annotations

Chapter 9

Implementing Responsible AI with AWS Services

Exam Focus: Understanding fairness, transparency, privacy principles and AWS tools like SageMaker Clarify and Bedrock Guardrails.

Responsible AI Principles

Fairness

  • Definition: Models should not discriminate against protected groups
  • Challenge: Historical bias in training data
  • Mitigation: Bias detection, fairness metrics, diverse datasets

Transparency & Explainability

  • Definition: Understanding how models make decisions
  • Importance: Trust, debugging, regulatory compliance
  • Approaches: Feature importance, SHAP values, attention visualization

Privacy

  • Definition: Protecting sensitive data and individual information
  • Techniques: Differential privacy, federated learning, encryption
  • Compliance: GDPR, CCPA, HIPAA requirements

AWS Tools for Responsible AI

SageMaker Clarify

Capability Purpose
Bias Detection Pre-training and post-training bias metrics
Feature Importance SHAP values for explainability
What-if Analysis Test model behavior under changes
Report Generation Comprehensive bias and explainability reports

Bedrock Guardrails

Feature Description
Content Filters Block harmful content categories
Denied Topics Prevent specific subjects
Word Filters Block specific words/phrases
PII Redaction Remove sensitive information
Contextual Grounding Check response relevance to source

Model Cards

  • Purpose: Document model characteristics and intended use
  • Content: Training data, performance metrics, limitations, risks
  • Benefit: Transparency and accountability

AI Security Fundamentals

Threats to AI Systems

Threat Description Mitigation
Prompt Injection Malicious input to manipulate output Input validation, guardrails
Data Poisoning Corrupting training data Data validation, monitoring
Model Extraction Stealing model through queries Rate limiting, output filtering
Membership Inference Determining if data was in training Differential privacy

AWS Security for AI

Service Purpose

KMS Encryption key management

PrivateLink Private connectivity
VPC Network isolation

Data Privacy & Protection

Encryption

Layer Implementation
At Rest KMS encryption for S3, databases
In Transit TLS 1.2+ for all communications
In Use Nitro Enclaves for sensitive processing

Access Control

  • IAM Policies: Role-based access to AI services
  • Resource Policies: Control access to models and endpoints
  • Condition Keys: Time-based, IP-based restrictions
  • VPC Endpoints: Private access without internet

Security Checklist

  • Enable encryption at rest and in transit
  • Implement least-privilege IAM policies

  • Enable CloudTrail logging
  • Implement guardrails for content filtering
  • Regular security audits and penetration testing

Exam Tips

Key Exam Strategies

  • Understand when to use managed AI services vs. building custom models
  • Know the differences between supervised, unsupervised, and reinforcement learning
  • Be able to select appropriate models based on use case requirements
  • Understand RAG architecture and when to use it
  • Know security best practices for AI workloads
  • Understand responsible AI principles and AWS tools for implementation

Common Exam Scenarios

Scenario Likely Answer
Need pre-trained API for text analysis Amazon Comprehend
Build Gen AI app without managing infrastructure Amazon Bedrock
Ground LLM responses in company data RAG with Bedrock Knowledge Bases
No-code ML for business users SageMaker Canvas
Monitor model performance in production SageMaker Model Monitor
Detect bias in ML models SageMaker Clarify
Content filtering for Gen AI Bedrock Guardrails

Key Terms to Remember

Learning Types

  • Supervised - Labeled data
  • Unsupervised - No labels
  • Reinforcement - Rewards/penalties
  • Self-supervised - Auto-generated labels

Gen AI Concepts

  • Tokens - Text units
  • Embeddings - Vector representations
  • Transformers - Attention-based architecture
  • Foundation Models - Pre-trained base models

Model Customization

  • Fine-tuning - Domain-specific training
  • Continued pre-training - Expand knowledge
  • Distillation - Compress large model
