Identity and Access Management (IAM)

IAM Fundamentals

Concept Definition
IAM User Long-term credentials for individuals or services
IAM Group Collection of users with shared permissions (cannot nest)
IAM Role Temporary credentials that can be assumed
IAM Policy JSON document defining permissions (Allow/Deny)

Policy Evaluation Logic

1. Default Deny (implicit)
2. Check for explicit Deny (overrides everything)
3. Check for explicit Allow
4. If Allow found â†’ Access granted
5. If no Allow â†’ Implicit Deny (access denied)

Chapter 11

Exam Preparation

Key Exam Strategies

  1. Identify the requirement: Security? Cost? Performance? Availability?
  2. Eliminate wrong answers: Look for impossible options first
  3. Compare remaining options: Choose the "best" answer
  4. Watch for keywords: "Most cost-effective", "highly available", "secure"

Common Answer Patterns

Scenario Likely Answer
Lowest cost, fault-tolerant Spot Fleet with Auto Scaling
Secure file sharing EFS with security groups
Global low-latency CloudFront with origin shield
Database HA Multi-AZ RDS with read replicas
Secrets rotation Secrets Manager with Lambda
DDoS protection Shield Advanced + WAF

ðŸ“ Final Exam Tips

  • Read the question completely before looking at answers
  • Identify what is being asked (security, cost, performance, availability)
  • Eliminate obviously wrong answers first
  • For "most cost-effective" questions, consider both compute and storage costs
  • For "highly available" questions, look for Multi-AZ, Auto Scaling, or Route 53 failover
  • For "secure" questions, look for encryption, IAM roles, or VPC isolation
  • You have 130 minutes for 65 questions - pace yourself (2 minutes per question)
  • Mark difficult questions for review and come back to them
  • No penalty for wrong answers - answer every question!
