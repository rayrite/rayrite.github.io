Designing Secure Access to AWS Resources

Pages 75-144 | IAM, IAM Policies, IAM Roles, AWS Organizations

🔑 IAM Components

ComponentPurposeBest Practice
UsersLong-term credentials for individualsUse for humans only; enable MFA
GroupsCollection of usersAttach policies to groups
RolesTemporary credentialsUse for applications, cross-account
PoliciesJSON documents defining permissionsLeast privilege principle
🧠 Mnemonic

"U=Unique person, G=Gather users, R=Relay pass, P=Permissions paper"

💡 Policy Evaluation Logic
  1. Explicit DENY → Always wins
  2. Explicit ALLOW → Required for access
  3. Implicit DENY → Default if no ALLOW

Key Rule: An explicit DENY always overrides an ALLOW

🔗 Connectivity Comparison
