AWS Cloud Adoption Frameworks Study Guide

The Four Phases of the Cloud Transformation Journey

Phase 1: Project

Core Purpose: The initial "toe in the water" phase to get hands-on experience by migrating a few, non-critical workloads, proving cloud value and building foundational skills.

Key Activities & Relevant AWS Services:
  • Experimentation: Launching a few EC2 instances.
  • Simple Storage: Using Amazon S3 to store backups or project files.
  • Cost Estimation: Using the AWS Pricing Calculator.
  • Basic Skills: Learning the AWS Management Console and setting up initial IAM users.

Phase 2: Foundation

Core Purpose: To build a secure, scalable, and operationally robust "landing zone." This is about creating the foundational infrastructure and governance for safe and efficient cloud adoption.

Key Activities & Relevant AWS Services:
  • Building a Landing Zone: Often automated with AWS Control Tower.
  • Centralizing Governance: Setting up AWS Organizations and using Service Control Policies (SCPs).
  • Establishing Identity: Using AWS Directory Service and AWS IAM Identity Center (SSO).
  • Creating a Secure Network: Designing a VPC strategy and connecting with AWS Direct Connect or VPN.
  • Automating Infrastructure: Creating templates with AWS CloudFormation.

Phase 3: Migration

Core Purpose: The "heavy lifting" phase to move a significant number of existing, on-premises applications and workloads to the cloud at scale.

Key Activities & Relevant AWS Services:
  • Portfolio Discovery: Using AWS Application Discovery Service.
  • Migration Planning & Tracking: Using AWS Migration Hub.
  • Server Migration: Using AWS Application Migration Service (MGN) to "rehost" servers.
  • Database Migration: Using AWS Database Migration Service (DMS).
  • Large-Scale Data Transfer: Using the AWS Snow Family or AWS DataSync.

Phase 4: Optimization & Reinvention

Core Purpose: The ongoing, mature phase focused on continuous improvement, cost optimization, and leveraging cloud-native services to reinvent business processes.

Key Activities & Relevant AWS Services:
  • Cost Optimization: Using AWS Cost Explorer, AWS Budgets, and AWS Compute Optimizer.
  • Performance Monitoring: Deeply using Amazon CloudWatch for observability.
  • Modernizing Architectures: Refactoring to microservices with Amazon ECS or AWS Lambda.
  • Leveraging AI/ML: Adding intelligence with services like Amazon Rekognition or Personalize.
  • Data Analytics: Building data warehouses with Amazon Redshift and dashboards with Amazon QuickSight.

The AWS CAF Cloud Adoption Journey Phases

1. Envision Phase

Core Purpose: To demonstrate how the cloud can accelerate business outcomes by identifying opportunities and creating a shared vision. Answers: "Why are we doing this, and what does success look like?"

Key Activities:
  • Identify Business Goals.
  • Envision the Future State.
  • Create the Business Case.

2. Align Phase

Core Purpose: To identify capability gaps across the organization and create a plan to address them, ensuring people, processes, and technology are ready. Answers: "Are we ready, and what do we need to build first?"

Key Activities:
  • Stakeholder Alignment.
  • Capability Assessment (using the six AWS CAF perspectives).
  • Create the Action Plan for the landing zone and upskilling.

3. Launch Phase

Core Purpose: To deliver the initial, pilot cloud workloads and demonstrate business value, learning and refining the process iteratively. Answers: "Let's start building and prove this works."

Key Activities:
  • Build the Landing Zone.
  • Execute Pilot Migrations.
  • Iterate and Learn from initial launches.

4. Scale Phase

Core Purpose: To expand production workloads and business value at scale, turning the launch process into a full-scale migration and modernization "factory." Answers: "How do we do this for the rest of the company?"

Key Activities:
  • Full-Scale Migration of remaining applications.
  • Decommission Old Systems to realize savings.
  • Operate and Optimize continuously.
  • Innovate with cloud-native services.
