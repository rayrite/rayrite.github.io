AWS Certified Cloud Practitioner (CLF-C02) Study Guide

Domain 1: Cloud Concepts (24%)

1.1 Define the AWS Cloud and its value proposition

  • Cloud Computing: On-demand delivery of IT resources over the internet with pay-as-you-go pricing.
  • Deployment Models:
    • Cloud: Fully on AWS.
    • On-premises (Private Cloud): Using virtualization and resource management tools in your own data center.
    • Hybrid: Connecting on-premises infrastructure to the cloud.
  • Types of Cloud Computing:
    • IaaS (Infrastructure as a Service): Basic building blocks for cloud IT (e.g., EC2). You manage the OS and applications.
    • PaaS (Platform as a Service): AWS manages the underlying infrastructure, you focus on deploying applications (e.g., Elastic Beanstalk).
    • SaaS (Software as a Service): A completed product managed by the provider (e.g., AWS Rekognition, Gmail).
  • Six Advantages of Cloud Computing:
    1. Trade capital expense (CAPEX) for operational expense (OPEX).
    2. Benefit from massive economies of scale.
    3. Stop guessing capacity.
    4. Increase speed and agility.
    5. Stop spending money on running and maintaining data centers.
    6. Go global in minutes.

1.2 Identify aspects of AWS Cloud economics

  • Total Cost of Ownership (TCO): Financial estimate to help identify direct and indirect costs of a system. The TCO Calculator helps compare the costs of running your applications in an on-premises or colocation environment to AWS.
  • Migration Strategies (The 7 Rs): Retire, Retain, Relocate, Rehost (lift-and-shift), Replatform (lift-and-reshape), Repurchase (drop-and-shop), Refactor/Re-architect.

1.3 Explain the different cloud architecture design principles

  • Well-Architected Framework: A set of best practices for cloud architects to build secure, high-performing, resilient, and efficient infrastructure for their applications.
  • The 6 Pillars:
    1. Operational Excellence: Run and monitor systems to deliver business value. (Perform operations as code, make small reversible changes).
    2. Security: Protect information, systems, and assets. (Implement strong identity, enable traceability, apply security at all layers).
    3. Reliability: Recover from disruptions and meet demand. (Test recovery, automatically recover from failure, scale horizontally).
    4. Performance Efficiency: Use computing resources efficiently. (Go global in minutes, use serverless architectures, experiment more often).
    5. Cost Optimization: Deliver business value at the lowest price. (Adopt a consumption model, measure efficiency, stop spending on data centers).
    6. Sustainability: Minimize the environmental impacts of running cloud workloads. (Understand your impact, maximize utilization, use managed services).

Domain 2: Security and Compliance (30%)

2.1 Define the AWS shared responsibility model

  • AWS's Responsibility (Security OF the Cloud): Protecting the infrastructure that runs all of the services offered by AWS. This includes hardware, software, networking, and facilities that run AWS services.
  • Customer's Responsibility (Security IN the Cloud): Management of guest OS, application software, firewall configuration (Security Groups), data encryption, and IAM.

2.2 Define AWS Cloud security and compliance concepts

  • IAM (Identity and Access Management): Manage access to AWS services and resources securely.
    • Users: End users (people, applications). Don't use the root user for daily tasks.
    • Groups: A collection of users to simplify permission management.
    • Roles: Assumed by trusted entities (users, services, applications) to gain temporary permissions.
    • Policies: JSON documents that define permissions (least privilege principle).
    • MFA (Multi-Factor Authentication): An extra layer of security for user logins.
  • AWS Artifact: On-demand access to AWS's security and compliance reports (e.g., PCI, SOC).
  • AWS Shield: Managed DDoS protection. Standard is free; Advanced provides additional protection.
  • AWS WAF (Web Application Firewall): Protects web applications from common web exploits (Layer 7) like SQL injection and XSS. Deployed on CloudFront, ALB, or API Gateway.
  • AWS KMS (Key Management Service): Managed service to create and control encryption keys used to encrypt data.
  • Amazon Inspector: Automated security assessment service for EC2, ECR container images, and Lambda functions to check for vulnerabilities and network accessibility.
  • Amazon GuardDuty: Intelligent threat detection service that continuously monitors for malicious activity using ML on CloudTrail, VPC Flow Logs, and DNS logs.
  • AWS Security Hub: A central place that aggregates, organizes, and prioritizes security alerts or findings from various AWS services.
  • Amazon Macie: A data security service that uses ML to discover, classify, and protect sensitive data (like PII) stored in S3.
  • AWS Config: Assesses, audits, and evaluates the configurations of your AWS resources. Tracks changes over time.
  • AWS CloudTrail: Logs all API calls made within your AWS account, providing an audit trail of actions.

Domain 3: Cloud Technology and Services (34%)

3.1 Define methods of deploying and operating in the AWS Cloud

  • AWS Management Console: Web-based user interface.
  • AWS CLI (Command Line Interface): Manage AWS services from the command line using access keys.
  • AWS SDK (Software Development Kit): Language-specific APIs to manage services programmatically.
  • AWS Elastic Beanstalk: PaaS for deploying and scaling web applications without worrying about the underlying infrastructure.
  • AWS CloudFormation: Infrastructure as Code service to model and provision all your cloud resources in a declarative template.

3.2 Define the AWS global infrastructure

  • Regions: A physical location in the world with multiple Availability Zones. Most services are region-scoped.
  • Availability Zones (AZs): One or more discrete data centers with redundant power, networking, and connectivity. They are isolated from each other.
  • Edge Locations / Points of Presence: Used by CloudFront to cache content closer to end users for lower latency.

3.3 Identify the core AWS services

  • Compute:
    • EC2 (Elastic Compute Cloud): Virtual servers (instances) in the cloud. Offers various instance types (General Purpose, Compute/Memory/Storage Optimized). Purchasing options include On-Demand, Reserved, Spot, and Dedicated.
    • Lambda: Serverless, event-driven compute service that lets you run code without provisioning servers.
    • Auto Scaling Group (ASG): Automatically adjusts the number of EC2 instances to maintain performance and optimize costs.
    • Elastic Load Balancing (ELB): Distributes incoming application traffic across multiple targets, such as EC2 instances. Types include Application (ALB), Network (NLB), and Gateway (GWLB).
    • ECS & Fargate: Run Docker containers. ECS requires managing EC2 instances, while Fargate is serverless.
  • Storage:
    • S3 (Simple Storage Service): Infinitely scalable object storage. Buckets must have a globally unique name. Key features include Storage Classes (Standard, IA, One Zone-IA, Glacier), Versioning, and Replication.
    • EBS (Elastic Block Store): Network-attached block storage for EC2 instances. Bound to a specific AZ.
    • EFS (Elastic File System): Managed network file system (NFS) that can be mounted on multiple EC2 instances across AZs.
    • S3 Glacier: Low-cost storage for data archiving and long-term backup.
    • AWS Storage Gateway: Hybrid service connecting on-premises storage to AWS cloud storage.
  • Networking:
    • VPC (Virtual Private Cloud): A logically isolated section of the AWS Cloud where you can launch AWS resources in a virtual network you define.
    • Route 53: Scalable Domain Name System (DNS) web service. Offers various routing policies (Simple, Weighted, Latency, Failover).
    • CloudFront: Content Delivery Network (CDN) that delivers content through a worldwide network of edge locations.
    • Direct Connect: Dedicated private network connection from on-premises to AWS.
    • AWS Global Accelerator: Improves application availability and performance by directing traffic over the AWS global network.
  • Databases:
    • RDS (Relational Database Service): Managed service for relational databases like MySQL, PostgreSQL, Oracle, etc. Supports Multi-AZ for high availability and Read Replicas for read scaling.
    • Amazon Aurora: AWS's proprietary relational database, compatible with MySQL and PostgreSQL, offering higher performance.
    • DynamoDB: Fully managed, serverless, key-value NoSQL database service with single-digit millisecond latency.
    • Redshift: Fast, fully managed data warehousing service for analytics (OLAP).
    • ElastiCache: In-memory caching service (Redis or Memcached) to improve performance of read-intensive applications.
  • Integration & Messaging:
    • SQS (Simple Queue Service): Fully managed message queuing service to decouple and scale microservices.
    • SNS (Simple Notification Service): Fully managed pub/sub messaging service.

3.4 Identify AWS technology support resources

  • AWS Trusted Advisor: Provides real-time guidance to help you provision your resources following AWS best practices across cost optimization, performance, security, fault tolerance, and service limits.
  • AWS Support Plans:
    • Basic: Free. Access to customer service, forums, and 7 core Trusted Advisor checks.
    • Developer: Business hour email access to Cloud Support Associates.
    • Business: 24x7 phone, email, chat access. Full Trusted Advisor checks. Sub-1-hour response for production down.
    • Enterprise On-Ramp & Enterprise: Includes a Technical Account Manager (TAM) and sub-15-minute response for business-critical down.

Domain 4: Billing, Pricing, and Support (12%)

4.1 Compare and contrast the various pricing models for AWS

  • Pay-as-you-go: Pay only for what you use, with no long-term contracts.
  • Save when you reserve:
    • Reserved Instances (RIs): Significant discount for a 1 or 3-year commitment on specific instance types.
    • Savings Plans: Flexible pricing model offering lower prices in exchange for a commitment to a consistent amount of usage ($/hour) for 1 or 3 years. Covers EC2, Fargate, and Lambda.
  • Pay less by using more: Volume-based discounts for services like S3 and data transfer.

4.2 Recognize the various account structures in relation to AWS billing and pricing

  • AWS Organizations: An account management service that enables you to consolidate multiple AWS accounts into an organization that you create and centrally manage.
  • Consolidated Billing: A feature of AWS Organizations that provides one bill for all accounts, volume pricing discounts, and sharing of reserved instances.
  • Service Control Policies (SCPs): Used in Organizations to manage permissions and restrict access to specific services for member accounts.

4.3 Identify resources available for billing support

  • AWS Pricing Calculator: Estimate the costs for your AWS architecture.
  • AWS Cost and Usage Report (CUR): The most comprehensive set of cost and usage data.
  • AWS Cost Explorer: A tool that lets you visualize, understand, and manage your AWS costs and usage over time.
  • AWS Budgets: Set custom budgets that alert you when your costs or usage exceed (or are forecasted to exceed) your budgeted amount.
  • AWS Cost Anomaly Detection: Uses machine learning to continuously monitor your cost and usage to detect unusual spends.
