AWS Shared Responsibility Model: CLF-C02 Study Guide

AWS Responsibility: Security OF the Cloud

AWS is responsible for protecting the infrastructure that runs all of the services offered in the AWS Cloud. This infrastructure is composed of the hardware, software, networking, and facilities that run AWS Cloud services.

  • Hardware & Global Infrastructure:
    • Regions, Availability Zones (AZs), and Edge Locations (Points of Presence).
    • Physical security of data centers (e.g., guards, fences, cameras).
  • Foundation Services (The "Undifferentiated Heavy Lifting"):
    • Compute: Physical servers, virtualization layer (hypervisor).
    • Storage: Physical disks, storage controllers.
    • Database: Physical hosts and infrastructure for RDS, DynamoDB, etc.
    • Networking: Physical routers, switches, cables.
  • Managed Services Patching: For services like RDS and DynamoDB, AWS manages the underlying OS and software patching.

Customer Responsibility: Security IN the Cloud

Your responsibility is determined by the AWS Cloud services that you select. This determines the amount of configuration work you must perform as part of your security responsibilities.

  • Customer Data: You own your data. You are responsible for its classification, protection, and encryption.
  • Platform, Applications, Identity & Access Management (IAM):
    • Managing users, groups, roles, and permissions using IAM.
    • Securing your application code and dependencies.
  • Operating System, Network & Firewall Configuration:
    • For IaaS (like EC2): You are responsible for patching the guest OS, installing antivirus, etc.
    • Configuring Security Groups (stateful firewall for instances).
    • Configuring Network Access Control Lists (NACLs) (stateless firewall for subnets).
  • Data Encryption & Integrity:
    • Client-Side Encryption: Encrypting data before sending it to AWS.
    • Server-Side Encryption (SSE): Configuring encryption for data at rest in services like S3, EBS, RDS.
    • Network Traffic Protection: Encrypting data in transit using TLS/SSL.

Exam Tips for CLF-C02

To correctly answer exam questions, ask yourself: "Can I configure this myself?"

  • If you can configure it in the AWS Console, CLI, or via API (e.g., an IAM policy, a Security Group rule, an S3 bucket policy), it is YOUR responsibility.
  • If it relates to the physical data center or the underlying hypervisor that runs EC2, it is AWS's responsibility.

Common Scenarios:

  • Patching an EC2 instance's Operating System? -> Customer Responsibility.
  • Patching the underlying host for RDS? -> AWS Responsibility.
  • Creating IAM users and assigning permissions? -> Customer Responsibility.
  • Ensuring physical data center security? -> AWS Responsibility.
  • Configuring an S3 bucket to be private? -> Customer Responsibility.
